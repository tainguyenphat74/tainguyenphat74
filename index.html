How I work

Build the small thing. Put it online. Repeat.

I'm Tai — a solo maker from Vietnam. I don't wait for a team, a budget, or permission.

Both products here started as something that annoyed me: a messy API doc I had to read anyway, and a website where I couldn't tell anyone what was broken without writing a paragraph about it.

So I built the fix, shipped it, and left it running for whoever else has the same problem. That's the whole method.
